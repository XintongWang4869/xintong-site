== Introduction


=== What is AsciiDoc

AsciiDoc is a lightweight markup language^1^ used for writing structured documentation, with an `adoc` file extension. It has easy-to-read syntax, rich features, and flexible customization options, making it ideal for creating technical documentation, books, and web content. With https://docs.asciidoctor.org/asciidoctor/latest/[Asciidoctor], a Ruby tool, it can be converted into multiple formats such as HTML, PDF, and EPUB.

[NOTE]
====
. _Markup language_: A human-readable language that describes how content should be displayed or structured. Other markup languages include HTML, Markdown, LaTeX, and XML.
====

=== AsciiDoc vs. Markdown
:hardbreaks-option:

Markdown is one of the most popular markup languages, commonly used in blog posts and is a conventional format for README files in software projects. It is easy to learn, enabling simple and quick content formatting, and has built-in support across various platforms including GitHub, GitLab, and static site generators. 

Due to its minimalistic design, it has become a favorite among general users, developers, and bloggers. However, compared to Markdown, AsciiDoc may be a better fit if you want to:

* *Equip your document with more features.*   
AsciiDoc natively supports advanced elements such as merging cells in tables and automatic cross-references between sections, without using extensions or plugins as Markdown does.

* *Customize the content and format more heavily.* 
AsciiDoc allows you to fine-tune the presentation by defining custom roles to apply specific CSS classes, or using attributes to dynamically insert reusable values across your document, which Markdown does not support.

* *Generate output in multiple formats.*
Using Asciidoctor, you can easily convert an AsciiDoc file to various formats including HTML, PDF, EPUB, DocBook, and slide decks, whereas Markdown requires additional tools or plugins.

== Writing AsciiDoc Documents
:hardbreaks-option:

With your editor set up, you can begin your AsciiDoc project. To generate polished outputs like PDF or HTML, you'll need supporting files such as YAML configuration and custom font settings, and it's important to organize your project properly. 

=== Document Structure Overview

Here is an example directory structure:
----
my-project/
├── content/
│   ├── index.adoc
│   └── chapters/
│       ├── introduction.adoc
│       └── setup.adoc
├── themes/
│   └── custom-theme.yml
├── fonts/
│   ├── roboto-regular.ttf
│   ├── roboto-bold.ttf
│   └── roboto-italic.ttf
└── images/
----

* *AsciiDoc Files* (`.adoc`) 
AsciiDoc files form the main body of your project. You can split a large project into multiple .adoc files and include them using the `include::` directive to keep things modular and manageable. For example, in `my-project/content/index.adoc`:
+
----
= My Project Title
Author Name
:doctype: book
:toc:
:sectnums:

\include::chapters/introduction.adoc[]
\include::chapters/setup.adoc[]
----

* *YAML Configuration Files (`.yml`)*
YAML files are often used alongside AsciiDoc when you use conversion tools like Asciidoctor PDF. The YAML theme file allows you to define project-level settings, such as page size, and font settings. You can use a theme that extends from the default AsciiDoc theme:
+
----
extend: default

page:
  size: A4
  layout: portrait

font:
  catalog:
    NotoSans:
      normal: fonts/NotoSans-Regular.ttf
      bold: fonts/NotoSans-Bold.ttf
      italic: fonts/NotoSans-Italic.ttf
      bold_italic: fonts/NotoSans-BoldItalic.ttf
  base:
    font_family: NotoSans
    font_size: 10

base:
  background_color: #ffffff
  text_color: #333333

table:
  border: 1px solid #ddd
  padding: 0.5em
  background: #fafafa
----

* *Customized Fonts*
By default, AsciiDoc uses system fonts, but you can specify your own font families for different parts of the document such as body text and code blocks. If your document includes non-English characters, you may need to reference a custom font to support these characters.

=== Images

==== To set fonts in SVG images

1. Create a file named [.underline]#xxx.cfg# and populate it with `skinparam defaultFontName Noto Serif` +
2. Pass the path to the corresponding attribute in adoc (or via the CLI or API) +
eg. `:xxxconfig: xxx.config`
3. Clear the cache of your diagrams and run Asciidoctor PDF with Asciidoctor Diagram enabled.

[NOTE]
====
You are free to choose a font name registered in your PDF theme; if using the default theme, you can choose "Noto Serif", "M+ 1mn", and "M+ 1p Fallback"
====

==== To scale images


pdfwidth - pt (default), % (of content area) ...  +
scale - percentage of the intrinsic width of the image +

[,yaml]
----
  image:
    width: 80%
    caption:
      end: bottom/top
      text-align: inherit
----

[NOTE]
====
* image-width -> takes precedence over the width attribute on an image macro
* image-caption-end -> caption location
* image-caption-text-align  -> can inherit the alignment of the image
====

==== To add background images

In adoc, define in the document header (or in the theme):

  :page-background-image: path/to/image
  :page-background-image-recto: path/to/image
  :page-background-image-verso: path/to/image[fit=none, pdfwidth=50%]
  :title-page-background-image: none

You can also place a logo image on the title page:

  :title-logo-image: image::path/to/image[top=25%,align=center,pdfwidth=0.5in]

=== Pages

==== To import pages

Import pages as import images (the first page is imported by default):

  image::xxx.pdf[page=3]  # import the third page  
  image::xxx.pdf[pages=2;10..16]  #import the second page, and page 10 to 16


==== To activate the title page

* Set `:doctype: book`
* Set `:title-page:` in the document header

=== To Generate TOC

Apply `:toc:` in the document title. +
Apply `:toclevels: <number>` in the document title to set levels of toc.

toc category keys (in yaml):

[,yaml]
----
  toc:
    text-transform: uppercase
    indent: 20
    line-height: 1.5
    break-after: auto
    h3-font-color: #999999
    h3-text-transform: none
    title:
      font-size: 18
      text-align: right
    dot-leader:
      content: "."
      font-style: bold
      levels: 2 3
      #if not specify dot-leader-levels, apply to all levels.
----

Click https://docs.asciidoctor.org/pdf-converter/latest/theme/toc/[here] to see the full list of keys.

=== PDF Outline

The outline is generated by default. The title and section levels can be customized.

Set in the document header: +
`:outline-title:` <preferred title>  +
`:outlinelevels: 3`  # can set between 1 to 5  +
`:outlinelevels: 4:2`  #section level depth: level depth

Set `[outlinelevels]` on the section's parent to override the `outlinelevels` document attribute

To disable the outline, apply `:!outline:`

=== Page Numbers

  page:
    numbering:
      start-at: after-toc

Other page-numbering-start-at values include `body` (default), `toc`, `cover`, `title`, and `integers`. See https://docs.asciidoctor.org/pdf-converter/latest/theme/page-numbers/[here] for difference.


[,yaml]
----
extends: default
footer:
  columns: = 100%
  recto:
    center:
      content: '(C) Copyright 2020 ACME. All rights reserved.'
    right:
      content:'{page-number}'
  verso:
    left:
      content: '{page-number} of {page-count}'
    center:
      content: $header-recto-center-content

----

To hide page numbers, do not include `{page-number}` in the footer


If content is the same, to reduce the amount of configuring: 

[,yaml]
----
  extends: default
  footer:
    columns: =100%
    recto: &shared_footer
      center:
        content: '{page-number}'
    verso: *shared_footer
----

=== Other Customization...

==== To prevent the orphaning of the anchor and title of a table, section, or discrete heading

Under the table heading, apply the breakable option: `[#optional-id%breakable]`


Above the section title: `[%breakable]`

If for all sections, set in the theme: 

  heading:
    min-height-after: auto


If you use [%unbreakable], then the whole content will be advanced to the next page if it will break otherwise.

==== To prevent wrapping the longest line

  [source%autofit,java]
  ----
  Here are codes. +
  The long line will not wrap, with decreased font size. 
  ----

To apply to the whole document: 

  = Document Title
  :autofit-option:


==== To hide section titles or numbers

Above the title, apply `[%notitle]` to hide the title: the title still shows up in the TOC.

Above the title, apply `:sectnums!:` to unset the number of this section.



==== To activate a syntax highlighter

`gem install rouge`

----
= Document Title
:source-highlighter: rouge
----


==== To customize figure/table captions
:hardbreaks-option:

Scenario: You want to use Figure X-X before the figure title for numbering.

----
== My Section
:doc-part:2

.My Image
image::{imagesdir}/my-image.png[caption="Figure {doc-part}-{counter:figure} "]
----

To reference figures/tables, you can add the attribute `xrefstyle` (full/basic/short). 
For example, for lists of figures/tables, you can use "full", but for main text, you can use "short".

You can also customize reference text using `reftext=`

==== Install Asciidoctor PDF (`asciidoctor-pdf`)

. Install ruby. Run `ruby -v` to check if ruby has been installed
. Run `gem install asciidoctor-pdf`

For more details, refer to https://docs.asciidoctor.org/pdf-converter/latest/

====
Asciidoctor PDF (gem: asciidoctor-pdf, command: asciidoctor-pdf) is published as a RubyGem to the rubygems.org software registry. You install Asciidoctor PDF from that registry using the RubyGem management tools provided by Ruby (bundle or gem). 
====

==== Install Fonts in WSL2

1. Update the package index on your Linux distribution by running the following command:   
`sudo apt update`

2. Install the fontconfig package by running the following command:   
`sudo apt install fontconfig`

3. Download the font files in either TTF or OTF format.

4. Copy the downloaded font files to the /usr/share/fonts/truetype directory in your Linux distribution. You can do this by running the following command:   
`sudo cp /path/to/downloaded/fonts/*.ttf /usr/share/fonts/truetype`

5. Update the font cache by running the following command:   
`sudo fc-cache -fv`

Run `fc-list` to check the list of installed fonts.
